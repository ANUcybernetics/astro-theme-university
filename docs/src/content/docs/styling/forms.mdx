The theme provides base styles for standard HTML form elements. No special classes are needed --- just use semantic HTML and the styles apply automatically.

## Text inputs

```html
<label for="name">Full name</label>
<input type="text" id="name" placeholder="Jane Smith" />
```

<Example label="Text input">
  <label for="demo-name">Full name</label>
  <input type="text" id="demo-name" placeholder="Jane Smith" />
</Example>

Other input types (`email`, `url`, `password`, `number`, `search`, `tel`, `date`) are styled identically.

## Select

Selects use a custom chevron indicator and `appearance: none` for cross-browser consistency.

```html
<label for="campus">Campus</label>
<select id="campus">
  <option>Acton</option>
  <option>Kioloa</option>
  <option>North Australia</option>
</select>
```

<Example label="Select">
  <label for="demo-campus">Campus</label>
  <select id="demo-campus">
    <option>Acton</option>
    <option>Kioloa</option>
    <option>North Australia</option>
  </select>
</Example>

:::info
Browsers are beginning to ship `appearance: base-select`, a CSS property that makes `<select>` elements fully styleable without JavaScript. It's stable in Chrome/Edge 135+ but not yet available in Firefox or Safari. Worth watching --- once support is broader it will replace the `appearance: none` workaround used here.
:::

## Checkboxes and radios

Checkboxes and radio buttons use the browser's native rendering with [`accent-color`](https://developer.mozilla.org/en-US/docs/Web/CSS/accent-color) set to the theme accent.

```html
<label> <input type="checkbox" /> I agree to the terms </label>
```

## Fieldset and legend

Group related fields with `<fieldset>` and `<legend>`.

```html
<fieldset>
  <legend>Contact details</legend>
  <label for="email">Email</label>
  <input type="email" id="email" />
  <label for="phone">Phone</label>
  <input type="tel" id="phone" />
</fieldset>
```

<Example label="Fieldset">
  <fieldset>
    <legend>Contact details</legend>
    <label for="demo-email">Email</label>
    <input type="email" id="demo-email" placeholder="jane@example.edu" />
    <div style="margin-block-start: var(--at-spacing-md);">
      <label for="demo-phone">Phone</label>
      <input type="tel" id="demo-phone" placeholder="+61 2 6125 5111" />
    </div>
  </fieldset>
</Example>

## Notes

Form styles live in the `at.base` CSS layer, so they apply to all form elements automatically. The styles are intentionally minimal --- they provide a consistent, accessible foundation that you can extend with your own CSS in a higher-priority layer.

Labels are `display: block` and bold by default. For inline labels (e.g. next to checkboxes), override with `display: flex` and `font-weight: normal` as shown in the checkbox examples above.
