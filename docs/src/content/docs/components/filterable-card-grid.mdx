The FilterableCardGrid component renders a list of items as cards with a searchbox above the grid. Typing into the searchbox filters the visible cards live by matching against the item's title, description, tags, and badges. It is designed as a drop-in replacement for `CardGrid` + `Card` on collection listing pages.

Each item can carry two kinds of metadata beyond its title and description: `tags` are searchable but invisible, while `badges` are short labels (e.g. `"Week 1"`) rendered as small chips between the title and description, and searched like everything else.

```astro
---
import FilterableCardGrid from "astro-theme-university/components/FilterableCardGrid.svelte";

const items = [
  { title: "Installation", href: "/install/", description: "Get up and running in minutes." },
  { title: "Configuration", href: "/config/", description: "Customise the theme to suit your site." },
  { title: "Deployment", href: "/deploy/", description: "Publish your site to the web." },
];
---

<FilterableCardGrid client:load {items} columns={2} label="Filter docs" />
```

## Keyboard shortcuts

- <kbd>/</kbd> --- focus the searchbox from anywhere on the page (ignored when
  typing in another input)
- <kbd>Escape</kbd> --- clear the searchbox while it is focused

## Filter behaviour

The query is split on whitespace into tokens. Every token must appear (substring match, case-insensitive) somewhere in the item's title, description, tags, or badges. With no query, all items are shown in their original order.

## Accessibility

- The searchbox uses `type="search"` with the supplied `label` as its `aria-label`.
- A live region announces the current match count whenever the query changes.
- The keyboard hint (`<kbd>/</kbd>`) is marked `aria-hidden`.
- Cards remain regular `<a>` elements --- focus order, hover states, and link behaviour match the standard `Card` component.

## Notes

Because this is a Svelte component, you must use a [`client:*` directive](https://docs.astro.build/en/guides/framework-components/#hydrating-interactive-components) to hydrate it. `client:load` is recommended so the searchbox is interactive immediately on page load.

Internal absolute paths (starting with `/`) in `href` are automatically prefixed with the site's base path, matching the behaviour of the `Card` component.
