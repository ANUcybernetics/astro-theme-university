The FilterableCardGrid component renders a list of items as cards with a searchbox above the grid. Typing into the searchbox filters the visible cards live by matching against the item's title, summary, and tags. It is designed as a drop-in replacement for `CardGrid` + `Card` on collection listing pages.

```astro
---
import FilterableCardGrid from "astro-theme-university/components/FilterableCardGrid.svelte";

const items = [
  { title: "Installation", href: "/install/", summary: "Get up and running in minutes." },
  { title: "Configuration", href: "/config/", summary: "Customise the theme to suit your site." },
  { title: "Deployment", href: "/deploy/", summary: "Publish your site to the web." },
];
---

<FilterableCardGrid client:load {items} columns={2} label="Filter docs" />
```

## Keyboard shortcuts

- <kbd>/</kbd> --- focus the searchbox from anywhere on the page (ignored when
  typing in another input)
- <kbd>Escape</kbd> --- clear the searchbox while it is focused

## Filter behaviour

The query is split on whitespace into tokens. Every token must appear (substring match, case-insensitive) somewhere in the item's title, summary, or any tag. With no query, all items are shown in their original order.

## Accessibility

- The searchbox uses `type="search"` with the supplied `label` as its `aria-label`.
- A live region announces the current match count whenever the query changes.
- The keyboard hint (`<kbd>/</kbd>`) is marked `aria-hidden`.
- Cards remain regular `<a>` elements --- focus order, hover states, and link behaviour match the standard `Card` component.

## Notes

Because this is a Svelte component, you must use a [`client:*` directive](https://docs.astro.build/en/guides/framework-components/#hydrating-interactive-components) to hydrate it. `client:load` is recommended so the searchbox is interactive immediately on page load.

Internal absolute paths (starting with `/`) in `href` are automatically prefixed with the site's base path, matching the behaviour of the `Card` component.
