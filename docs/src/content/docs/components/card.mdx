The Card component displays content in a contained, titled block. When an `href` is provided the entire card renders as an `<a>` element, making the whole surface clickable. Without `href`, it renders as a `<div>`.


## Notes

When using the `image` prop, provide meaningful `imageAlt` text if the image conveys information. The default empty string marks the image as decorative, which is appropriate for purely visual imagery but not for informational graphics.

When cards are placed inside a [CardGrid](/docs/components/card-grid/), they use [CSS subgrid](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_grid_layout/Subgrid) to align their image, title, and body rows across all cards in the same grid row. This keeps cards visually aligned even when titles or body text vary in length.
